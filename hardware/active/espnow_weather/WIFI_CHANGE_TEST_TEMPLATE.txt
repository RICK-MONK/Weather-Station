WIFI CHANGE TEST TEMPLATE

IMPORTANT
- The screenshot shows `MonaConnect` on `5 GHz` with channel `56`.
- The network security type is `Open`, so the ESP32 password should be an empty string: `""`.
- A standard ESP32 gateway cannot use `5 GHz` Wi-Fi directly.
- This will still work only if your router also exposes `MonaConnect` on `2.4 GHz`.

CHANNEL CHECK FOR ESP-NOW

After the gateway joins the new `2.4 GHz` Wi-Fi, open the Serial Monitor and look for:

Wi-Fi channel: X

Then make sure both of these files use the same `ESPNOW_WIFI_CHANNEL` value:

- `display_espnow_weather_gateway/display_espnow_weather_gateway.ino`
- `sensor_espnow_weather_sender/sensor_espnow_weather_sender.ino`

Example:

#define ESPNOW_WIFI_CHANNEL 6

QUICK TEST CHECKLIST

1. Start the backend on your computer so it is listening on port `5000`.
2. Confirm your computer IP on the new Wi-Fi and update the `BACKEND_IP_*` octets once.
3. Make sure the Wi-Fi network is `2.4 GHz`, not `5 GHz`.
4. Flash the gateway and check for `Wi-Fi connected`.
5. Check the printed Wi-Fi channel and set the sender/gateway `ESPNOW_WIFI_CHANNEL` to match.
6. Flash the sender and confirm you get sensor packets plus a successful HTTP POST.

IF WIFI CHANGES AGAIN

- The two values you will usually need to change are the Wi-Fi name and the `BACKEND_IP_*` octets.
- If the new network uses a different `2.4 GHz` channel, update `ESPNOW_WIFI_CHANNEL` in both sketches.
